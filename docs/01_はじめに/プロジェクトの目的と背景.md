# プロジェクトの目的と背景

## 1. プロジェクトの概要
本プロジェクトは、青葉福祉会における人事考課を対象としたWEBアプリケーションの設計・実装を目的として開始した。  
具体的には、考課者の段階的な評価、評価基準の管理、評価結果の集計・可視化、といった人事考課の業務フローを情報システムとして一貫して扱える仕組みの構築を目指した。

本来であれば、業務システムとして実運用可能な水準まで完成させることが理想であったが、実装期間や経験値不足などから、最終的にアプリケーション全体を完成させるに至らなかった。  
一方で、本プロジェクトでは、単なる試作にとどまらず、実務レベルの設計思想・技術選定・構造化、つまり長期運用可能性の高さを重視して開発を進めており、その過程で得られた知見や反省点は、後続の開発者や同様のテーマに取り組む者にとって有用であると考えられる。

そのため、本資料では**設計と実装に至るまでの思考過程と判断の記録**を成果として整理することを目的とする。

## 2. このプロジェクトに取り組んだ背景
本アプリケーションに着手した背景には、二つの問題意識がある。

第一に、人事考課にかかる作業量と運用上の負担が非常に大きい点である。  
現状では、紙に直接記入する方式やスプレッドシートを用いた運用が中心となっており、考課者側での入力ミスや合計スコアの計算ミスが、各事業所単位で発生する可能性がある。これらのミスは考課結果の信頼性を下げるだけでなく、本部側での確認や修正作業を必要とし、業務全体の負荷を増大させている。  
特に人事考課は複数の評価項目や考課者が関与する業務であるため、集計や確認に一定の正確性が求められるが、現行の運用では人手によるチェックに依存せざるを得ず、ミスの完全な防止が難しいという構造的な問題を抱えている。

第二に、人事考課が十分に有効活用されていない点である。  
本来、人事考課は単なる評価のための仕組みではなく、被考課者の成長や育成につなげるための重要なツールである。しかし現状では、評価結果の振り返りが体系的に行われておらず、被考課者自身が自分の評価結果を確認できない、あるいは過去の評価履歴を追跡できないといった状況が見られる。  

以上の背景から、本プロジェクトでは、人事考課業務を単にデジタル化するのではなく、入力ミスや集計ミスを抑制しつつ、評価結果を継続的に活用できる仕組みを設計・実装することを目指してアプリケーション開発に取り組んだ。

## 3. プロジェクトの目的
本プロジェクトの目的は、人事考課業務を単に電子化・効率化することではなく、人事考課を継続的な人材育成のための仕組みとして活用できる基盤を作ることにあった。

まず、業務運用の観点では、考課者による入力ミスや集計ミスを可能な限り抑制し、事業所ごとの差異や属人的な処理を減らすことを目指した。  
評価項目や計算ルールをシステム側で一元管理することで、考課結果の正確性と公平性を担保し、本部側での確認・修正作業にかかる負担を軽減することが、第一の理想像である。

次に、被考課者の視点からは、人事考課の結果を自ら振り返り、成長に生かせる情報として蓄積・参照できる状態を実現することを目標とした。具体的には、被考課者自身が評価結果を確認でき、過去の履歴を時系列で追うことができることで、自身の強みや課題を継続的に認識できる仕組みを想定していた。

また、考課者および施設長や役員にとっても、人事考課が個別の業務に閉じるのではなく、組織全体の育成やマネジメントに活用できる情報基盤になることを目指した。  
評価結果を集計・分析しやすい形で管理することで、特定の評価項目における傾向把握や、育成施策の検討に活用できる状態が理想である。

これらを踏まえ、本プロジェクトにおける理想像は、以下の三点に集約される。
- 考課者の入力・集計ミスを抑制し、業務負担を軽減すること
- 被考課者が評価結果を振り返り、成長に活かせる仕組みを提供すること
- 評価データを組織として蓄積し、人材育成に活用できる基盤を整えること

本アプリケーションは、これらの状態を実現するための手段として設計・実装されたものであり、以降の章では、この理想像をどのような設計方針・技術選定によって実現しようとしたのかを説明する。
